Version 2.9.1 is out

 
 

08/05/2019

2.9.1 (May 8, 2019)

LANGUAGE

  1. •support for input or output-less actors (see for ex. examples/misc/ioless and examples/misc/ioless2 and Sec. 2.4.4, 2.4.7 of the LRM)

  2. •redefinition of the foldl and foldli higher-order primitives, which now respectively have type ('a * 'a -> 'a) -> 'a -> 'a bundle -> 'a and (unsigned -> 'a * 'a -> 'a) -> 'a -> 'a bundle -> 'a (see updated examples in examples/network/{foldl,foldli,foldmap} and examples/fir/fir1n)

  3. •added higher-order primitive chain, similar to pipe but giving access to all intermediate values (see examples/network/chain and example/fir/fir1n)

TOOLS

  1. •caphmake now (optionally) takes the name of the project file (.proj) as argument

  2. •caphmake now systematically generates Makefiles in sub-directories (./dot, ./sim, ./system and ./vhdl) so that subsequent processing can be carried out by simply cd'ing to the corresponding subdir and invoking the various targets

GUI

  1. •the location of the CAPH library is now part of the .ini file and the .cphpro project file

  2. •the Build SystemC and Build VHDL actions now also generates a Makefile for subsequent compilation

EXAMPLES

  1. •added examples [dpn/{ex1-4,ex1-8,ex3-5}], from T. Parks PhD to illustrate the transcription of DPNs (Dataflow Process Networks) in CAPH

  2. •directories/files have been reorganized/rewritten to match the new directory scheme of caphmake

DOC

  1. •Updated sections 2.4.4 and 2.4.7 for describing the definition and use of IO-less actors resp.

  2. •Updated section 2.4.7.5 on higher-order primitives with foldl, foldli and chain
