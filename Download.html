The CAPH framework is provided both as source code and as pre-built installers for MacOS and Windows platforms.


Building the compiler from source distribution requires Objective Caml and a C/C++ compiler (preferably GCC). Building the GUI additionnally requires Qt (vers >=5.2).


The Windows (resp. Mac OS) distribution provides pre-compiled, 32 (resp. 64) bit, native executables.


In both cases, some extra packages may be needed (they are not part of the Caph distribution)

  1. -to visualize the data-flow graphs generated by the compiler, you will need the graphviz package.

  2. -for compiling the SystemC generated code, a working SystemC installation is required (we use the one distributed by Logic Poet

  3. -for simulating and synthetizing the VHDL generated code, a simulation and synthesis tool chain (we use GHDL for simulation and Quartus II from Altera for synthesis). Float support for the VHDL backend requires the  FPHDL package
